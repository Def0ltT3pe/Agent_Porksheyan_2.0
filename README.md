# Agent Porksheyan

> Система распределённого мониторинга и выполнения задач с использованием агентов, очереди задач и REST API.

---

## 📋 Описание проекта

Проект представляет собой систему, в которой центральный сервер управляет агентами, распределяет задачи и собирает результаты их выполнения.

Агенты подключаются к серверу, получают задачи из очереди, выполняют их (например, сбор системной информации) и отправляют результат обратно.

---

## 🏗️ Архитектура

Система построена по распределённой архитектуре и состоит из следующих компонентов:

| Компонент | Описание |
|---|---|
| **FastAPI сервер** | Управление агентами и задачами |
| **База данных (SQLAlchemy)** | Хранение состояния |
| **Redis** | Очередь задач |
| **Agent (Python)** | Исполнитель задач |

```
Agent → FastAPI → Database
           ↓
         Redis
```

---

## ⚙️ Основная функциональность

### Агенты
1. Регистрация агента на сервере
2. Аутентификация по токену
3. Отправка heartbeat (проверка активности)

### Задачи
1. Создание задач через API
2. Очередь задач через Redis
3. Назначение задач агентам
4. Отслеживание статусов:
   - `pending`
   - `running`
   - `completed`

### Выполнение
1. Получение задачи агентом
2. Выполнение (например, сбор системной информации)
3. Отправка результата на сервер

---

## 🗄️ База данных

Для работы с базой данных используется ORM SQLAlchemy.

### Основные сущности

**Agents**
| Поле | Описание |
|---|---|
| `id` | Уникальный идентификатор |
| `name` | Имя агента |
| `token` | Токен аутентификации |
| `status` | Статус (`online` / `offline`) |
| `last_seen` | Время последнего heartbeat |

**Tasks**
| Поле | Описание |
|---|---|
| `id` | Идентификатор задачи |
| `agent_id` | Привязка к агенту |
| `type` | Тип задачи |
| `payload` | Входные данные (JSON) |
| `status` | Статус (`pending` / `running` / `completed`) |
| `created_at` / `started_at` / `finished_at` | Временные метки |
| `result` | Результат выполнения (JSON) |
| `logs` | Логи выполнения |

> База данных хранит состояние системы, а Redis используется как очередь задач.

---

## 🔄 Принцип работы

1. Агент регистрируется на сервере
2. Сервер выдаёт токен
3. При создании задачи:
   - она сохраняется в БД
   - добавляется в Redis-очередь
4. Агент получает задачу из Redis
5. Выполняет её
6. Отправляет результат обратно
7. Сервер обновляет статус задачи

## 🚀 Установка и запуск

### 1. Установка зависимостей

```bash
pip install fastapi uvicorn sqlalchemy redis requests psutil python-dotenv
```

### 2. Запуск Redis

```bash
redis-server
```

### 3. Запуск сервера

```bash
cd server
uvicorn main:app --reload
```

- Сервер доступен по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)
- Swagger документация: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

### 4. Запуск агента

```bash
cd agent
python agent.py
```
